# PLAYBOOK — PDF оқулықтан офлайн жаттығу сайтын жасау

Бұл құжат — осы сессияда не істелгенінің толық жазбасы. Мақсаты: **дәл осындай
сайтты басқа кітапқа қайта жасау**. Ішінде өнімнің сипаттамасы, дерек құбырының
(pipeline) алгоритмдері, схема, баптау константалары және жаңа кітапқа көшу
тізімі бар.

Бұл жердегі нақты сайт: *English Vocabulary in Use, Pre-Intermediate (Cambridge,
4th ed.)* — 100 unit, 471 тапсырма, 2752 item.

# 1-БӨЛІМ. Өнім (сайт)

## 1.1 Негізгі талап

Толығымен офлайн: сервер жоқ, build қадамы жоқ, интернет жоқ. `index.html`-ды
қос шертіп ашасың. Пайдалану сценарийі: **оң жақта PDF, сол жақта сайт**.
Суреттер мен кестелер PDF-те қалады, сайт сұрақ мәтінін көрсетеді.

## 1.2 Файл құрылымы

```
index.html      бет қаңқасы (bare skeleton, логика жоқ)
style.css       стильдер, light/dark автоматты (prefers-color-scheme)
app.js          барлық логика (бір IIFE, тәуелділік жоқ)
data.js         window.EVU_DATA = {...}  ← exercises.json-нан генерацияланады
exercises.json  дерек
tools/          PDF-тен деректі құратын скрипттер + регрессия тесті
```

> **Неге `data.js`?** `file://` арқылы ашқанда `fetch()` CORS себебімен
> істемейді. Дерек `<script>` арқылы тікелей жүктелуі керек. Генерациялау:
> ```
> python3 -c "import json;d=json.load(open('exercises.json',encoding='utf-8'));open('data.js','w',encoding='utf-8').write('window.EVU_DATA = '+json.dumps(d,ensure_ascii=False,separators=(',',':'))+';')"
> ```

## 1.3 Беттер (hash routing)

| Маршрут | Не көрсетеді |
|---|---|
| `#/unit/N` | Unit беті: барлық тапсырмалар, әр item-де өріс + тексеру |
| `#/errors` | Қате жіберілген, әлі меңгерілмеген item-дер (unit бойынша топталған, сол жерде қайта жауап беруге болады) |
| `#/stats` | Жалпы карточкалар + 100 unit бойынша кесте + прогресті өшіру |

Тұрақты элементтер: жоғарыда 3 таб (қателер санымен badge), сол жақта іздеуі бар
unit тізімі (әрқайсысында прогресс %), мобильде sidebar жиналады.

## 1.4 Item түрлері және рендері

| Тип | Не істейді |
|---|---|
| `items` | Негізгі: сұрақ + өріс + «Тексеру» (автоматты) |
| `text` | Нөмірленген бос орындары бар **тұтас мәтін**: мәтін жоғарыда бір рет, астында нөмірленген өрістер |
| `open` | «Over to you» — жауап кілті жоқ, өзің ✓/✗ белгілейсің |
| `freeform` | Санатқа бөлу / сөз тізімі: «Жауапты көрсету» батырмасы, тексеру жоқ |
| `crossword` | Тек ескерту: торды PDF-тен шеш |

Қосымша: сәйкестендіру жаттығуларында (`Match the words…`) оң жақтағы a/b/c
нұсқалары **бөлек тізім** болып бір рет көрсетіледі (`options` өрісі).

Item деңгейіндегі белгілер:
- `isExample` — кітаптың дайын мысалы: өріс жоқ, мәтін бірден көрінеді, прогреске кірмейді.
- `exampleAnswers` немесе `answer: null` — автоматты тексеру мүмкін емес, өзің белгілейсің.
- Егер `answer` нормаланғанда бос болса (`""`, `"–"`) — ол да өзің тексеретінге өтеді.

## 1.5 Жауапты тексеру алгоритмі

Бұл ең маңызды бөлік. Реті:

1. **`norm(s)`** — кіші әріп → тырнақша/апостроф бір түрге → **әріп пен саннан
   басқасының бәрін алып тастау**. `"He's tying"` → `hestying`.
   Нәтиже: үтір, нүкте, артық бос орын кедергі емес.
2. **`expandParens(s)`** — `(...)` міндетті емес: `eat (any) lunch` → `eat lunch`
   **және** `eat any lunch`. Рекурсивті, 64 нұсқамен шектелген.
3. **`expandSlashTokens(s)`** — сөз ішіндегі `/` бір сөзді алмастырады:
   `took/got a train` → `took a train`, `got a train`. (Декарттық көбейтінді, 64-пен шектеу.)
4. **`splitAlternatives(s)`** — фраза деңгейінде `/` және бөлек тұрған ` or `:
   `comes out /is published` → екі бөлек нұсқа.
5. **`buildVariants(answer)`** — 2–4-ті біріктіріп, барлық қабылданатын
   нормаланған нұсқалар жиынын құрайды (түпнұсқа жол да кіреді).
6. **`isMatch(input, answer)`** — терілгенді нормалап, жиында бар-жоғын тексереді.

**Әдейі бөлінбейтіні:** `;` — ол бір емес, **бірнеше бос орынның** жауабы
(`degrees; zero; extremely`). Бөлсең, толық емес жауап дұрыс деп саналар еді.

Бос жол терілсе — тексеру жүрмейді (өріске фокус қана қойылады).

Кез келген шеткі жағдайға сақтандырғыш: қате шыққанда **«Мен дұрыс жаздым»**
батырмасы жауапты дұрысқа ауыстырады.

## 1.6 Прогресс моделі

- `localStorage`, кілт `evu_v1`, құрылым:
  `{v:1, items:{"<unit>|<sub>|<n>": {streak, wrong, last, mastered, val, self}}}`
- `MASTER_STREAK = 3` — қатарынан 3 дұрыс → `mastered`, «Қателер» тізімінен шығады.
- Қате → `streak=0`, `wrong++`, `mastered=false`.
- `val` — терілген мәтін (тексерілмесе де сақталады, debounce ~120мс).
- `self` — оқушы өзі белгілегенін білдіреді.
- Прогресс пайызы: тек **жауап кілті бар** түрлер (`items` + `text`) есептеледі.
  `open`/`freeform` пайызды төмендетпейді.

## 1.7 UX ұсақ-түйектері (бірақ маңызды)

- **Enter** — жолды тексеріп, келесі өріске фокусты жібереді.
- Тексерілген соң: жол жасыл/қызыл/алтын (меңгерілген) болады, астында кітаптың
  жауабы («Кітап: …») және streak («2/3 қатарынан») көрсетіледі.
- Әр тапсырманың соңында «Тапсырманы тексеру», unit соңында «Барлығын тексеру».
- Unit бетінде PDF бет нөмірі чип болып тұрады (жаттығу беті + теория беті).

# 2-БӨЛІМ. Дерек құбыры (PDF → JSON)

## 2.1 Басты түсінік

**Дайын JSON-ға сенбе — PDF-тен қайта құр.** Бізге берілген JSON-да 500-ден астам
қате болды, себебі PDF-тен қарапайым (`get_text()`) оқылған. Кітап макеті мұны
көтермейді:

1. **Бағаналы (multi-column)** — PyMuPDF жолдарды солдан оңға қарай қатарлап
   береді, сондықтан екі бағанадағы мәтін бір-бірімен араласады.
2. **Нөмір оң жақ шетте** — «Complete the text» түріндегі жаттығуларда item
   нөмірі оң жақ шетте (x≈505), сөйлем сол жақта (x≈149). Қарапайым оқығанда
   нөмір мен сөйлем **бір-біріне жылжып** қиысады → жауаптар түгел бір орынға
   ауысады. (Бұл ескі JSON-дағы ең үлкен қате көзі болды.)
3. **Теория мен жаттығу бір бетте** («Study units») — теория мәтіні сұраққа сіңеді.

Шешім: **координатамен жұмыс істеу**. Барлық логика жолдардың `x`/`y`
координатасына сүйенеді.

## 2.2 Скрипттер

```
tools/pdflib.py        ортақ: PDF ашу, жолдарды координатамен алу, бағанаға бөлу
tools/extract_key.py   кітап соңындағы Answer key → tools/key.json
tools/extract_ex.py    жаттығу беттері → tools/ex_raw.json
tools/build.py         екеуін біріктіру → exercises.json
tools/test.html        браузердегі регрессия тесті (43 тексеру)
```

Тәуелділік: `pip3 install pymupdf` (тек деректі құруға керек, сайттың өзіне емес).

Жүргізу реті:
```
python3 tools/extract_key.py
python3 tools/extract_ex.py
python3 tools/build.py
python3 -c "…"          # data.js генерациялау (1.2-бөлімде)
```

## 2.3 Негізгі принцип: жауап кілті — басым дереккөз

**Сұрақтар** жаттығу беттерінен, **жауаптар** кітап соңындағы Answer key-ден
алынады. Нөмірлеу дауласқанда **Answer key жеңеді**, себебі ол таза нөмірленген
тізім болып басылған (әр жауап өз нөмірімен басталады) — оны сенімді талдауға
болады. Жаттығу беті әлдеқайда шатақ.

## 2.4 Answer key талдау (`extract_key.py`)

1. «Answer key» деп басталатын бірінші бетті тап → сол жерден Index-ке дейін оқы.
2. Әр беттегі тақырыптар: жолдың мәтіні **дәл** `6.3` немесе `Unit 7` болса — ол
   тақырып (кәдімгі мәтін ешқашан тек содан тұрмайды).
   ⚠️ **Тұзақ:** тақырыпты сол шетке байлама. Кейбір бетте `x=0`-де жалған жол
   болады да, шет есептеу бұзылады; кейбір бетте тақырып x=73, кейбірінде x=106.
3. Тақырыптар арасындағы жолдарды ал, **бағанаға бөліп** (x бойынша кластер),
   әр бағананы y бойынша сұрыпта → нағыз оқу реті.
4. Әр жолды `^(\d{1,2})\s+(.*)$` деп талда: нөмір — жаңа item, нөмірсіз жол —
   алдыңғысының жалғасы.

Нәтиже: 473 тапсырма, 2396 нөмірленген жауап.

## 2.5 Жаттығу беттерін талдау (`extract_ex.py`)

**Аймақ (region) анықтау.** Әр `N.M` тақырыбы үшін: сол тақырыптан келесі
тақырыпқа дейінгі, әрі `x >= тақырып.x - 6` жолдар. Соңғысы маңызды — «Study
unit»-те тақырып оң бағанада тұрса, сол жақтағы теория автоматты түрде шығып
қалады.

**Теорияны кесу.** Аймақ келесі теория бөлімінің белгісіне дейін ғана созылады.
Бөлім белгісі = жалғыз бас әріп (`A`, `B`, `C`…) өлшемі ≥16.

**Жолдарды талдау (`parse_region`)** — реті:
1. `rows_of()` — жолдарды **y бойынша қатарға** топта (tolerance 7px), әр қатарды
   x бойынша сұрыпта.
2. Item нөмірі болатын сегменттерді тап (`^(\d{1,2})(\s+(.*))?$`, 1..30).
3. Олардың x-ін кластерле. **≥2 кластер, әрқайсысында ≥2 нөмір болса** — бұл
   екі бағаналы жаттығу; әйтпесе бір ағын.
4. Item құрастыру:
   - **Екі бағаналы:** бағана шекарасы — кластер орталықтарының ортасы. Әр
     сегмент өз бағанасының ағымдағы item-іне жазылады.
   - **Бір ағын:** қатарда нөмір қай жерде тұрса да (сол жақта да, оң шетте де)
     сол қатарды белгілейді → *off-by-one мәселесі осында шешіледі*.
   - **Көлденең тізім:** бір қатарда `«N сөз»` түріндегі ≥2 сегмент болса
     (`1 knee  2 comb  3 castle`), әрқайсысы бөлек item.
5. Нұсқаулық = бірінші item қатарына дейінгі жолдар.

**Мысал item-дерді табу.** Кітап дайын жауаптарды **қолжазба қаріппен**
(`CalibanStd`) басады. Сол қаріп кездескен item = мысал. Бұл эвристикадан әлдеқайда
дәл. Қосымша: Answer key `2`-ден басталса, `1` — мысал (кітап мысалды кілтке
енгізбейді) — бұл қаріп ұстамаған жағдайларды жабады.

**Нұсқаулық vs мәтін.** Нұсқаулық **қалың** (Bold), мәтін кәдімгі қаріппен.
«Complete the text» жаттығуларында нұсқаулық пен мәтін бір-біріне жабысады —
сонда қалың жолдардың соңы бойынша бөлінеді (`bold_split`).

**Сәйкестендіру нұсқалары.** `^([a-h])\s+(.+)$` түріндегі, бір x-те тұрған,
`a`-дан басталатын кем дегенде 3 сегмент табылса — бұл нұсқалар бағанасы.
Оларды бөлек `options` тізіміне шығарып, item мәтінінен алып тастау керек.

**Футерді сүзу.** «English Vocabulary in Use…» жолы + беттің төменгі жағындағы
таза сандық жол (бет нөмірі).

## 2.6 Біріктіру және жіктеу (`build.py`)

Тип анықтау реті **маңызды**:

```
1. crossword     нұсқаулықта "crossword" бар
2. Over to you   нұсқаулықта "Over to you" бар  → items (кілт болса) / open / freeform
3. text          "passage_like": кілттегі жауап саны ≥3, бірақ табылған item саны
                 кілттің 70%-нан аз ЖӘНЕ мәтін ұзындығы >150 таңба
4. freeform      item мүлде табылмаса
5. items         қалғаны
```

⚠️ **«Over to you» міндетті түрде 3-тен бұрын тексерілуі керек** — оның үлгі
жауаптары ұзын сөйлемдер болғандықтан, «мәтін» деп қате танылады.

⚠️ **«Over to you» жауаптары ешқашан автоматты тексерілмейді.** Кітаптағысы —
бір адамның үлгі жауабы. Бәріне `exampleAnswers: true` қойылады. (Әйтпесе
оқушының өз жауабы «қате» деп белгіленеді.)

Тағы: кілтте `Possible answers:` / `Your own answers` префиксі болса — сол да
`exampleAnswers`.

**Unit тақырыбы** — теория бетінің жоғарғы жағындағы ең үлкен жазу (өлшемі
25–40, y<70). Ескі JSON-дағы тақырыптарда сурет жазуы жабысып қалған еді
(«Cooking barbecue frying pan» → «Cooking»).

**Бет нөмірлері** — `pdfPages` = тапсырмалар кездескен беттер жиыны.
Теория беті: бір бетте болса `pp[0]-1`, бірнеше бетте болса (Study unit) `pp[0]`.

**Есеп.** `build.py` `tools/report.txt` жазады — қолмен қарауды қажет ететін
жерлер. Бізде 5 қалды, бәрі суретке қарап жауап беретін жаттығулар (мәтіні жоқ —
бұл дұрыс).

## 2.7 JSON схемасы

```jsonc
{
  "book": "…",
  "units": [{
    "unit": 7,
    "title": "Weather",
    "pdfExercisePage": 22,       // 1-based PDF беті
    "pdfPages": [22],            // тапсырмалар кездесетін барлық бет
    "pdfIntroPage": 21,
    "subExercises": [{
      "number": "7.1",
      "type": "items",           // items | text | open | freeform | crossword
      "instructions": "Match the words on the left with the words on the right.",
      "options": [{"letter":"a","text":"with rain"}],   // сәйкестендіруде ғана
      "passage": "…1 … 2 …",                            // type=text болғанда
      "rawQuestion": "…", "rawAnswer": "…",             // type=freeform болғанда
      "stub": true, "note": "…",                        // type=crossword болғанда
      "items": [{
        "n": 2,
        "question": "heavy",     // null болуы мүмкін (сурет / бос орын)
        "answer": "f",           // null болуы мүмкін
        "isExample": true,       // міндетті емес
        "exampleAnswers": true   // міндетті емес
      }]
    }]
  }]
}
```

# 3-БӨЛІМ. Жаңа кітапқа көшіру

## 3.1 Кітапқа тәуелді константалар

Бәрі `tools/` ішінде. Жаңа кітапта **алдымен осыларды тексер**:

| Константа | Қайда | Осы кітапта | Не істейді |
|---|---|---|---|
| `PDF` | `pdflib.py` | — | PDF жолы |
| `HAND_FONT` | `extract_ex.py` | `CalibanStd` | Дайын мысал жауаптың қаріпі |
| `HEAD_RE` + `size>=15, bold` | `extract_ex.py` | `^\d+\.\d+$` | Тапсырма тақырыбы |
| `SECTION_RE` + `size>=16` | `extract_ex.py` | `^[A-H]$` | Теория бөлімінің белгісі |

| `CROSS_RE` | `build.py` | `crossword` | Кроссворд |
| бағана tolerance | `pdflib/extract_ex` | 45px | x кластерлеу |
| қатар tolerance | `extract_ex.py` | 7px | y топтау |

## 3.3 Жаңа кітапқа арналған тізім

1. `pdflib.py`-дағы `PDF` жолын ауыстыр.
2. **Барлау:** бір жаттығу бетінің және бір Answer key бетінің жолдарын
   координатасымен басып шығар. Қаріптерді сана. 3.1-кестедегі константаларды
   салыстыр.
3. `extract_key.py` жүргіз → `key.json`-дағы 3-4 тапсырманы кітаппен салыстыр.
4. `extract_ex.py` жүргіз → tapсырма саны кілттегімен шамалас па?
5. `build.py` жүргіз → `tools/report.txt` оқы. Ондағы әр жазба — не шынымен
   суретті жаттығу, не баптау қате.
6. Автоматты аудит жүргіз (3.4).
7. **PDF-тің 3-4 бетін суретке айналдырып, көзбен салыстыр** (міндетті!).
8. `data.js` генерациялап, `tools/test.html`-ды жүргіз.

## 3.4 Автоматты аудит (әр құрудан кейін)

Мыналарды сана — бәрі 0-ге немесе күтілгенге жақын болуы керек:

- сұрақта футер/«Answer key» қалдығы
- 180 таңбадан ұзын сұрақтар (теория сіңіп кеткен белгісі)
- нұсқаулығы бос тапсырмалар
- мысал емес, бірақ жауабы жоқ item-дер
- бір тапсырмада қайталанатын сұрақ мәтіні
- кілтте бар, бірақ сұрағы жоқ item-дер (`report.txt`)
- «Over to you» ішінде автоматты тексерілетін item (**0 болуы шарт**)
- әр жауаптың өз-өзімен сәйкес келуі (`isMatch(ans, ans) == true`)

## 3.5 Тест

```
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" --headless \
  --allow-file-access-from-files --virtual-time-budget=40000 \
  --user-data-dir=/tmp/evu-test --dump-dom "file://$PWD/tools/test.html" \
  | sed -n '/===TESTS===/,/===END===/p'
```

⚠️ Chrome headless `--dump-dom`-нан кейін өз-өзінен жабылмайды — фонда жүргізіп,
`sleep` + `kill` қажет. Нәтижені **синхронды** жаз (setTimeout ішінде ғана
жазсаң, virtual-time бітіп қалуы мүмкін). Скриптті `try/catch`-пен ора, әйтпесе
бір қате бүкіл нәтижені жоғалтады.

## Жол бойы табылған тұзақтар (қайталанады)

1. **Off-by-one** — нөмір оң шетте тұратын жаттығулар. Ең көп зиян келтіргені.
2. **Answer key тақырыбын сол шетке байлау** — `x=0`-дегі жалған жол бұзады.
3. **«Over to you» автоматты тексерілуі** — оқушының жауабы қате деп саналады.
