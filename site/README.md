# Ағылшын — жаттығулар

Cambridge-тің алты оқулығының жаттығулары бір сайтта: басты бет (таныстыру +
кітап таңдау) → юнит тапсырмалары. Пайдаланушыға арналған нұсқаулық — `#/help`
бетінде.

## Құрылымы

```
site/
  index.html            қабық: home / book көріністері + нұсқаулық терезесі
  style.css             тема (vocab-preint тақырыбы + кітап карточкалары)
  app.js                бүкіл логика — vocab-preint қозғалтқышы, кітапқа жалпыланған
  i18n.js               интерфейс мәтіндері: kk / en
  help.js               «Қалай қолдану» нұсқаулығы (терезе ретінде ашылады), kk / en
  dict.js               сөз аудармасы: меңзеу / қос шерту / белгілеу / ұзақ басу
  books.js              кітаптар каталогы (атауы, деңгейі, түсі, PDF жолы)
  data/index.json       әр кітаптың юнит/сұрақ саны — welcome және library беттеріне
  data/<id>.json        юниттер (сұраныс бойынша жүктеледі)
  data/dict.json        сөздік ядросы (tools/dict_core.tsv-ден жиналады)
  pdf/<id>.pdf          оқулықтар
  tools/build_data.py   бастапқы қалталардан data/ жасайды
  tools/dict_core.tsv   сөздіктің қолмен жазылған ядросы
  tools/build_dict.py   dict_core.tsv → data/dict.json
```

Түпнұсқа алты қалта (`../vocab-preint/`, `../enginuse/`, …) өзгертілген жоқ —
олар осы сайттың бастапқы деректері болып қала береді.

## Деректерді қайта жинау

```
python3 site/tools/build_data.py
```

Алты кітапты біркелкі схемаға келтіреді:

```
{unit, title, section?, pdfExercisePage?, pdfPages?, pdfIntroPage?,
 subExercises: [{number, type, instructions?, note?, wordBank?, options?,
                 passage?, rawQuestion?, rawAnswer?,
                 items: [{n, question?, answer?, isExample?, exampleAnswers?}]}]}
```

`section` — кітаптың Contents-індегі бөлім (мыс. «Modals»). grammar мен
vocab-upint-те бар; статистика бетінде бөлім бойынша жинақ көрсетіледі.
grammar атаулары мен бөлімдері `tools/grammar_titles.json`-нан, vocab-upint
бөлімдері `tools/vocab_upint_sections.json`-нан алынады.

`type`: `items` | `text` | `open` | `freeform` | `crossword`.

Прогреске тек `items` және `text` типтеріндегі, мысал емес сұрақтар кіреді —
бұл ереже `app.js` ішіндегі `isTracked()` пен `build_data.py` ішіндегі
`tracked()` функцияларында бірдей болуы керек.

## Сөздік (dict.js)

Жаттығудағы кез келген ағылшын сөзін меңзесең (немесе қос шертсең, белгілесең,
телефонда ұзақ бассаң) — қазақша аудармасы мен қарапайым ағылшынша түсініктемесі
шығады. Іздеу екі көзден жүреді:

1. `data/dict.json` — қолмен жазылған ядро, 1385 сөз. Лезде, интернетсіз
   жұмыс істейді және оқулықтардағы сөз қолданысының ~86%-ын жабады.
2. Қалғаны — тегін API-лар (Google Translate `gtx` + dictionaryapi.dev). Бір
   рет алынып, `localStorage`-ке (`agylshyn_wl_cache_v1`) сақталады, келесіде
   жергіліктідей жылдам шығады. Мұндай жауап «интернеттен · машина аудармасы»
   деп белгіленеді — сапасы ядродағыдай емес.

`dict.js` сөздің түбірін өзі табады: `neighbours → neighbour`, `bigger → big`,
`stopped → stop`, `isn't → is`. Сондықтан TSV-ге тек негізгі форманы жазу
жеткілікті; дұрыс емес формалар (`went`, `children`) ғана бөлек жазылады.

Ядроға сөз қосу:

```
# tools/dict_core.tsv, бағандар табпен бөлінеді:
# сөз <TAB> қазақша <TAB> қарапайым ағылшынша <TAB> сөз табы (міндетті емес)
kitten	марғау	kid of a cat	noun

python3 tools/build_dict.py                 # қайта жинау + жабу пайызы
python3 tools/build_dict.py --missing 200   # келесі не қосу керегін көрсетеді
```

`build_dict.py` ішіндегі `base_forms()` — `dict.js` ішіндегі `baseForms()`
көшірмесі. Біреуін өзгертсең, екіншісін де қатар өзгерт: ол тек жабу пайызын
дұрыс санау үшін керек.

## Жергілікті іске қосу

`fetch()` қолданылатындықтан файлды жай екі рет басып ашуға болмайды — сервер керек:

```
cd site && python3 -m http.server 8777
# http://localhost:8777
```

## Жариялау (GitHub Pages)

`site/` қалтасының ішіндегісін репозиторийге салып, Pages-ті сол бұтаққа
бағыттау жеткілікті. Барлық жолдар салыстырмалы, сондықтан ішкі қалтада да
(`user.github.io/repo/`) жұмыс істейді.

Ескерту: `pdf/` — 147 МБ, ең үлкені `essential-grammar.pdf` (71 МБ).
GitHub-тың бір файлға шегі 100 МБ, сондықтан өтеді, бірақ репозиторий ауыр
болады. PDF қажет болмаса, `pdf/` қалтасын алып тастап, `books.js` ішіндегі
`pdf` өрістерін өшірсең — бет нөмірлері жай мәтін болып қалады.

## Беттер

| Hash | Не көрсетеді |
|---|---|
| `#/` | Басты бет: таныстыру, статистика, кітап карточкалары, нұсқаулық түймесі |
| `#/help` | Басты бетті ашып, нұсқаулық терезесін бірден шығарады |
| `#/b/<id>` | Кітаптың бірінші юниті |
| `#/b/<id>/unit/<n>` | Нақты юнит |
| `#/b/<id>/errors` · `#/b/<id>/stats` | Қателер / статистика |

Ескі `#/books` сілтемесі басты бетке апарады.

## PDF беттерінің нөмірленуі

Бастапқы алты қалтадағы бет нөмірлері әртүрлі болжаммен жазылған, сондықтан
`build_data.py` ішіндегі `PDF_PAGE_OFFSET` арқылы нақты PDF бетіне түзетіледі:

| Кітап | Ығысу |
|---|---|
| essential-grammar | 0 |
| grammar | +1 |
| advanced-grammar | +11 |
| vocab-preint | 0 |
| vocab-upint | 0 |
| vocab-adv | +2 |

Ығысулар PDF-тің өз мәтінінен әр юниттің жаттығу нөмірлерін (`1.1`, `1.2`…)
іздеу арқылы өлшенген. **PDF ауыстырылса, оларды қайта өлшеу керек.**

Алты кітаптың бәрінде юнит екі беттен тұрады: сол жақта — түсініктеме,
оң жақта — жаттығу. Сондықтан `pdfIntroPage` жоқ жерде ол `pdfExercisePage - 1`
деп есептеледі. «PDF-ті ашу» түймесі түсініктеме бетінен ашады, ал екі чип
(«Түсініктеме беті», «Жаттығу беті») өз бетін ашады.

## PDF панелі және өлшемді өзгерту

Юнит бетіндегі «PDF-ті ашу» түймесі кітапты сол беттің оң жағында, `<iframe>`
ішінде ашады — жаңа терезе де, жаңа қойынды да ашылмайды. Панельдер арасында
екі сүйрегіш (`.dragger`) бар: сол жақтағысы юниттер тізімінің, оң жақтағысы
PDF панелінің енін өзгертеді. Ені `--sidebar-w` / `--pdf-w` айнымалылары
арқылы қойылады да, `localStorage`-тағы `ui` өрісінде сақталады. Сүйрегішті
қос шертсең — бастапқы қалпына оралады.

**Маңызды:** Chrome-ның PDF көрсеткіші жүктеліп болған соң `src`-тің тек
фрагментін (`#page=`) ауыстырғанды елемейді — бет ауыспай, сол қалпы тұрып
қалады. Сондықтан `mountPdf()` бет өзгерген сайын `<iframe>` элементінің
өзін жаңасымен алмастырады. Оны жеңілдетуге тырыспа.

`&navpanes=0` сияқты қосымша параметрлер PDF-ті мүлдем жүктетпей қояды —
тек `#page=N` қолдан.

## Тақырып (ашық / қараңғы)

Оң жақ жоғарыдағы дөңгелек түйме екі күйді ауыстырады: `light` (☀) ↔ `dark` (☾).
«Жүйе бойынша» деген үшінші күй жоқ. Таңдау `localStorage`-та `theme` өрісінде
сақталады, ал `<html data-theme="light|dark">` **әрқашан** қойылып тұрады.

Жүйенің параметрі (`prefers-color-scheme`) тек бір рет — бірінші кіргенде,
бастапқы мәнді таңдау үшін оқылады. Одан кейін оның өзгеруі ештеңеге әсер
етпейді. `style.css` ішінде қараңғы палитра бір-ақ жерде:
`:root[data-theme="dark"]`.

`index.html` ішіндегі `<head>`-тегі шағын скрипт тақырыпты бет боялғанға дейін
қояды (сақталғаны болмаса — жүйе бойынша) — сондықтан ақ жарқыл болмайды.
CSS-те `prefers-color-scheme` сақтандырғышы жоқ, сол себепті бұл скрипт
атрибутты міндетті түрде қоюы керек.
Ондағы `agylshyn_v1` кілті `app.js`-тегі `STORE_KEY`-мен бірдей болуы керек.

## Тіл / Language

Интерфейс екі тілде: қазақша (kk) және ағылшынша (en). Оң жақ жоғарыдағы
**KK / EN** ауыстырғышы үш беттің бәрінде бар. Таңдау `localStorage`-та
сақталады; бірінші кіргенде браузер тілі бойынша анықталады (ағылшын болса
en, әйтпесе kk).

Барлық мәтін `i18n.js` ішінде, `kk` және `en` кілттері бірдей болуы керек.
Жаңа жол қосқанда екеуіне де қос — `t()` жетпеген кілтті `kk`-дан, ол да
болмаса кілттің өзін көрсетеді, сондықтан олқылық бірден көрінеді.

Кітап сипаттамалары (`blurb`) — `books.js` ішінде, екі тілде.
Жаттығулардың мазмұны аударылмайды: ол оқулықтың өз мәтіні (ағылшынша).

The interface is bilingual — Kazakh (kk) and English (en), switchable with the
**KK / EN** control in the top-right corner of every page. The choice is stored
in `localStorage` and defaults to the browser language on a first visit.
UI strings live in `i18n.js`; exercise content itself stays in English, since
that is the coursebook text.

## Прогресс

Барлығы бір `localStorage` кілтінде: `agylshyn_v1`.
Жазба кілті — `bookId|unit|subNumber|itemNumber`, сондықтан кітаптар бір-біріне
кедергі жасамайды. Статистика бетіндегі «өшіру» түймесі тек сол кітапты өшіреді.

Ескі қосымшалардың прогресі (`evu_v1`, `egu_v1`, `agiu.progress.v2`, …)
көшірілген жоқ — олар өз кілттерінде сақтаулы тұр.

## Белгілі мәселе

`essential-grammar` PDF-і мәтін қабатысыз сканерленген, сондықтан бастапқы
шығарылымда юнит атаулары толық танылмаған: 114 юниттің 43-і «Unit N» болып
тұр. Атаулар ойдан жазылмайды — дұрысы керек болса, PDF-ті OCR арқылы қайта
өңдеу керек.
